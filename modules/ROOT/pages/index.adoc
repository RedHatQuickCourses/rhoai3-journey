= Industrialized AI Token Platform — Journey Map
:navtitle: Journey Map
:toc: macro
:description: Single learning journey to deploy an industrialized AI token generation platform on Red Hat OpenShift AI—optimized resources, performance SLOs, model governance, and business value.

[.lead]
*One course. One narrative. From governed data and models to a production token factory with validation.*

This journey combines nine enablement modules into a single path for technical associates who want to deploy an **industrialized AI token generation platform** on Red Hat OpenShift AI 3.x. The focus is *value*: optimized resources, performance SLOs, model governance, and engineering outcomes—not depth for its own sake.

== Who This Is For

* **Audience:** Internal associates with a technical background (platform engineers, software engineers, ops)—not necessarily Generative AI experts.
* **Prerequisites:** OpenShift/RHOAI access, `oc` CLI, basic Kubernetes (namespaces, pods, secrets). GenAI concepts are introduced in context.

== How to Use This Site

* Follow the phases in order; each builds on the previous.
* From each phase page, use the links into the source modules for full labs and deep dives.
* **Engineering checkpoints:** Data Connection + PVC → Model from Catalog → Hardware profile in use → InferenceService Ready → llm-d stack / Arcade lab → MaaS endpoint with API key → Validation job and report.
