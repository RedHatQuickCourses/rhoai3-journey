Getting Started with RHOAI3 Journey

🎯 What is the RHOAI3 Journey?

The RHOAI3 Journey is a unified learning path that combines 10 separate training courses into one comprehensive guide for deploying an industrialized AI platform on Red Hat OpenShift AI 3.x.

It's not a single repository—it's a multi-repository architecture where one "journey map" repo orchestrates content from 10 sibling course repos.

Prerequisites

  • Claude Code installed on your system
  • Git and GitHub account configured
  • Code editor (VS Code, vim, or similar)
  • Basic command-line familiarity
  • Node.js 16+ (for local builds)
  • Docker (optional, for containerized builds)

Understanding the Architecture

The journey consists of 1 journey repository + 10 course repositories:

parent/
├── rhoai3-journey/ ← Journey map (navigation & structure)
├── rhoai3-operators/ ← Phase 0: Operators
├── rhoai3-storage/ ← Phase 1: Storage
├── rhoai3-registry/ ← Phase 1: Model Registry
├── rhoai3-catalog/ ← Phase 1: Model Catalog
├── rhoai3-hwprofiles/ ← Phase 1: Hardware Profiles
├── rhoai3-gpu-aas/ ← Phase 1: GPU as a Service
├── rhoai3-deploy/ ← Phase 2: Model Serving
├── rhoai3-llmd/ ← Phase 3: Distributed Inference
├── rhoai3-maas/ ← Phase 3: Models as a Service
└── rhoai3-validate/ ← Phase 4: Validation

💡 Key Concept: Separation of Concerns

Journey repo (rhoai3-journey): Contains only navigation structure, phase introductions, and wrapper pages that link to course content.

Course repos (rhoai3-storage, etc.): Contain the actual training content—presentations, labs, quizzes.

Understanding Antora & Course Structure

🎯 What is Antora?

Antora is a static site generator designed specifically for creating documentation sites from AsciiDoc files stored across multiple git repositories.

It's the engine that powers the RHOAI3 Journey and all Red Hat QuickCourses.

Why Antora for Training Content?

📦 Component-Based

Each course is a separate "component" that can be versioned, built, and published independently.

🔗 Multi-Repository

Aggregate content from multiple git repositories into a single unified site.

📚 Content Reuse

Include and reference content across components using xref and include directives.

🎨 Consistent UI

Single UI bundle applied across all courses for brand consistency.

Antora Terminology

Term Definition Example
Component A discrete unit of documentation (typically one course/repo) rhoai3-storage
Module A subdivision of a component (usually by topic/chapter) chapter1, ROOT
Page An individual AsciiDoc file that becomes an HTML page section1.adoc
Playbook Configuration file that defines what content to build antora-playbook.yml
Component Descriptor Metadata file for a component antora.yml
UI Bundle Theme/template that controls site appearance ui-bundle.zip

Standard Antora Repository Structure

Single Course Repository (e.g., rhoai3-storage)
rhoai3-storage/
├── antora.yml # Component descriptor
│ ├── name: rhoai3-storage # Component identifier
│ ├── title: Course Title # Display title
│ ├── version: 3.3 # Version number
│ └── nav: [...] # Navigation files
│
├── modules/ # All content goes here
│ ├── ROOT/ # Default/root module
│ │ ├── nav.adoc # Navigation for ROOT
│ │ ├── pages/ # AsciiDoc content files
│ │ │ └── index.adoc # Landing page
│ │ ├── images/ # Images for ROOT module
│ │ ├── examples/ # Code examples
│ │ └── attachments/ # Downloadable files
│ │
│ └── chapter1/ # Additional module (chapter)
│ ├── nav.adoc # Chapter navigation
│ ├── pages/
│ │ ├── index.adoc # Chapter landing
│ │ ├── section1.adoc # Content pages
│ │ ├── section2.adoc
│ │ └── lab.adoc
│ └── images/ # Chapter-specific images
│
├── antora-playbook.yml # Build config (for standalone builds)
├── package.json # NPM dependencies
└── .github/workflows/ # CI/CD automation

How Antora Builds the Site

  1. Read the Playbook

    Antora starts with antora-playbook.yml, which defines:

    • Which repositories to include
    • UI bundle location
    • Output directory
    • Site metadata
  2. Clone Content Sources

    For each repository listed in the playbook:

    • Clones the repo (or uses local path)
    • Reads its antora.yml to understand the component
  3. Parse Content

    For each component:

    • Scans modules/ directories
    • Processes AsciiDoc files
    • Resolves includes and cross-references
  4. Generate Navigation

    Builds navigation menus from nav.adoc files

  5. Apply UI Bundle

    Wraps content in the theme/template

  6. Output HTML

    Generates the static site in build/site/

Key Antora Concepts

1. Content References (xref)

Link to pages within or across components:

// Same component
xref:section2.adoc[Next Section]

// Different component
xref:rhoai3-storage:chapter1:index.adoc[Storage Guide]

// Pattern: component:module:page.adoc[link text]

2. Content Inclusion (include)

Embed entire pages or snippets:

// Include from another component
include::rhoai3-storage:chapter1:section1.adoc[]

// Include from same component
include::partial$snippet.adoc[]

3. Resource References

// Images (from same module)
image::architecture.png[Architecture Diagram]

// Images (from another module)
image::chapter1:gpu-layout.png[]

// Attachments
link:{attachmentsdir}/config.yaml[Download Config]

4. Navigation Files

Navigation files (nav.adoc) use a simple list syntax:

// modules/ROOT/nav.adoc

* xref:index.adoc[Home]
* xref:chapter1:index.adoc[Chapter 1]
** xref:chapter1:section1.adoc[Section 1]
** xref:chapter1:section2.adoc[Section 2]
* xref:chapter2:index.adoc[Chapter 2]

The RHOAI3 Journey Pattern

How Multi-Repository Aggregation Works

10 Course Repos
(Content)
→
Journey Playbook
(Aggregator)
→
Single Unified Site
(Output)

Step by step:

  1. Each course repo is built independently (for testing)
  2. Journey playbook lists all 10 course repos as content sources
  3. Journey repo creates "wrapper pages" that include content from courses
  4. Single build produces unified site with all content

Example: Storage Content Flow

Content Source (rhoai3-storage repo):
modules/chapter1/pages/section1.adoc
↓
Wrapper Page (rhoai3-journey repo):
modules/ROOT/pages/p1-storage-section1.adoc
contains: include::rhoai3-storage:chapter1:section1.adoc[]
↓
Navigation (rhoai3-journey):
modules/ROOT/nav2.adoc
contains: *** xref:p1-storage-section1.adoc[Storage Architecture]
↓
Generated Site:
build/site/rhoai3-journey/1/p1-storage-section1.html

Where to Edit What

To Change... Edit This File... In This Repo...
Course content (text, labs, quizzes) modules/chapter1/pages/*.adoc Course repo (e.g., rhoai3-storage)
Journey navigation order modules/ROOT/nav2.adoc rhoai3-journey
Phase introduction pages modules/ROOT/pages/phase*.adoc rhoai3-journey
Which courses are included antora-playbook.yml rhoai3-journey
Component name or version antora.yml Each course repo
Site-wide theme/appearance ui-bundle/ rhoai3-journey

Common Antora Build Commands

Build locally (course repo)
npx antora antora-playbook.yml
Build with remote repos (journey)
npx antora --fetch antora-playbook.yml
Watch mode (auto-rebuild)
npm run watch:adoc
Serve locally
npm run serve

Troubleshooting Antora Issues

Error: "Page not found" or "Unresolved xref"

Causes:

  • File doesn't exist at the specified path
  • Component name mismatch (check antora.yml)
  • Module name typo

Solution: Verify the xref pattern matches the actual structure

Error: "Component not found"

Causes:

  • Repository not in playbook content sources
  • Sibling repo not cloned (for local builds without --fetch)

Solution: Add repo to playbook or use --fetch flag

Navigation doesn't update

Causes:

  • Cache issue
  • Wrong nav file referenced in antora.yml

Solution: Delete build/ and rebuild

Common Content Patterns

Including External Content (Wrapper Pages)

Journey wrapper pages use the include:: directive to pull in course content:

// File: rhoai3-journey/modules/ROOT/pages/p1-storage-section1.adoc
:navtitle: Storage Architecture

include::rhoai3-storage:chapter1:section1.adoc[]

Pattern: include::<component>:<module>:<file>.adoc[]

  • <component> = component name from course's antora.yml
  • <module> = module directory (usually chapter1)
  • <file> = path to file within module's pages directory

Cross-Referencing Between Courses

GitHub Workflows Explained

Main Branch Workflow (main.yml)

Triggers: Push to main branch

What it does:

  1. Checks out the repository
  2. Installs Node.js and dependencies
  3. Runs npx antora --fetch antora-playbook.yml
  4. Publishes to GitHub Pages

📦 The --fetch Flag

The --fetch flag tells Antora to clone all 10 course repos from GitHub (via git URLs in the playbook). This ensures the full journey is built even though only the journey repo is checked out.

Pull Request Workflow (pr.yml)

Triggers: PR opened, reopened, synchronized, or closed

What it does:

  1. Builds a preview of the site with PR changes
  2. Publishes preview to a unique URL
  3. Removes preview when PR is merged/closed

💡 PR Previews

PR previews only work for branches in the same repository, not from forks. This is a security measure to prevent malicious code execution.

Submitting an Issue

  1. Navigate to the correct repository

    Journey issues: https://github.com/RedHatQuickCourses/rhoai3-journey/issues

    Course issues: https://github.com/RedHatQuickCourses/rhoai3-storage/issues (example)

  2. Click "New Issue"
  3. Provide clear title and description

    Good title: "Storage section 2 PVC command fails on OCP 4.15"

    Bad title: "Broken"

    Include:

    • What you expected
    • What actually happened
    • Steps to reproduce
    • Environment (if relevant)

Submitting a Pull Request

  1. Fork or branch the repository

    For external contributors: Fork the repo

    For team members: Create a branch

    Terminal
    git checkout -b fix/storage-pvc-update
  2. Make your changes

    Edit files, test locally, ensure build succeeds

  3. Commit with clear message
    Terminal
    git add modules/chapter1/pages/section2.adoc
    git commit -m "Fix PVC creation command for OCP 4.15"
  4. Push to your fork/branch
    Terminal
    git push origin fix/storage-pvc-update
  5. Create PR on GitHub

    Navigate to the repo, click "Compare & pull request"

    PR description should include:

    • What changed
    • Why it changed
    • How to test
    • Related issues (if any)

Best Practices

  • Test locally before submitting PR
  • Keep PRs focused (one fix per PR)
  • Write clear commit messages
  • Reference related issues (Fixes #123)
  • Respond to review comments promptly
  • Ensure CI builds pass

7-Step Integration Process

  1. Ensure your course repo has proper structure

    Your new course must have:

    rhoai3-newcourse/
    ├── antora.yml # Must have: name, title, version, nav
    ├── modules/
    │ ├── ROOT/
    │ │ ├── nav.adoc
    │ │ └── pages/index.adoc

    │ ├── pages/
    │ │ ├── index.adoc
    │ │ ├── section1.adoc
    │ │ └── ...
    │ └── images/

    Check your antora.yml:

    name: rhoai3-newcourse        # ← IMPORTANT: Used in include:: directives

- modules/ROOT/nav.adoc
- modules/chapter1/nav.adoc

Component Naming Conventions

📝 Critical: Component Name Consistency

The name: field in your course's antora.yml MUST match exactly what you use in include:: directives.

Example:

  • In rhoai3-newcourse/antora.yml: name: rhoai3-newcourse
  • In wrapper page: include::rhoai3-newcourse:chapter1:index.adoc[]

Include and Xref Patterns

Pattern Use Case Example
include:: Embed entire page include::rhoai3-newcourse:chapter1:index.adoc[]
xref: Link to another page xref:p5-newcourse-home.adoc[]
xref: (external) Link to other component xref:rhoai3-storage:chapter1:index.adoc[Storage Guide]

Troubleshooting Integration

Build Error: "Component not found"

Cause: Component name mismatch

Solution: Check that antora.yml name matches include directive

Navigation Doesn't Show New Phase

Cause: Wrapper pages don't exist or nav not updated

Solution:

  • Verify wrapper pages created in modules/ROOT/pages/
  • Check nav2.adoc has xref entries
  • Ensure filenames match exactly (case-sensitive)

Content Doesn't Display

Cause: Include path incorrect

Solution: Verify pattern: component:module:file.adoc

  • Component = name: from antora.yml
  • Module = directory name (e.g., chapter1)
  • File = path within pages/

Local Build Works, CI Fails

Cause: Sibling repo not in playbook git URLs

Solution: Ensure your repo is listed in antora-playbook.yml with full GitHub URL

Common Errors and Solutions

Error: "antora: command not found"

Solution: Run npm install first

Error: "Page not found" in navigation

Solutions:

  • Check filename matches xref exactly (case-sensitive)
  • Verify file exists in pages/ directory
  • Check for typos in navigation file

Error: Build succeeds but content blank

Solutions:

  • Check include pattern: component:module:file.adoc
  • Verify component name in antora.yml
  • Ensure sibling repo is cloned (local) or in playbook (CI)

Watch mode not rebuilding

Solutions:

  • Kill and restart npm run watch:adoc
  • Check you're editing files in modules/
  • Try manual build: npm run build
