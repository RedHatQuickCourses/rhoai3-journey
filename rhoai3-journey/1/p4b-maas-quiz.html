Knowledge Check: Industrializing AI (OpenShift AI 3.3)

Did you master the factory floor? Let’s find out.

You have explored the strategy, mastered the taxonomy, built the GitOps factory, and learned how to keep it running. Test your understanding with these 6 scenarios.

Question 1: The Value Proposition (Business Case)

Scenario: Your organization is debating whether to build an internal MaaS platform or use only public APIs (e.g., OpenAI) for enterprise LLM workloads. Which benefit does MaaS provide that public APIs typically do not?

  • A. Lower compute costs at very high, unpredictable usage spikes.

  • B. The ability to completely avoid managing any infrastructure.

  • C. Guaranteed SLOs (predictable latency) and mathematical control over where proprietary data is processed.

  • D. Immediate access to the absolute largest models in the world without any setup.

Click here for the Answer

Correct Answer: C
MaaS gives you control over your capacity and governance. It allows you to guarantee Service Level Objectives (SLOs) without "noisy neighbors" and keeps proprietary data entirely within your private network (Zero Trust). Public APIs share capacity and expose data externally.

Question 2: The Architecture Choice

Scenario: Your enterprise needs to provide an internal, ChatGPT-like API for 500 developers across 10 different departments. You need to enforce token quotas to charge each department for their specific usage. Which inference architecture must you choose in OpenShift AI 3.3?

  • A. A standard InferenceService using the standard vLLM runtime.

  • B. An LLMInferenceService using the Distributed Inference Server (llm-d) runtime.

  • C. The Caikit-TGIS standalone runtime.

Click here for the Answer

Correct Answer: B
In OpenShift AI 3.3, true MaaS capabilities—including the Gen AI Studio UI, API key generation, and tier-based token quotas—are exclusively supported by the Distributed Inference Server (llm-d). Standard vLLM is used for standalone, isolated workloads that do not require multi-tenant governance.

Question 3: Network Optimization

Question: You are deploying a massive model (e.g., Llama-3-70B), which requires tensor parallelism across multiple physical worker nodes. To prevent the network from bottlenecking the GPU-to-GPU communication, what Technology Preview feature should your Cluster Administrator enable?

  • A. KEDA Autoscaling

  • B. GPU Time-Slicing

  • C. RoCE (RDMA over Converged Ethernet)

  • D. The Gateway API

Click here for the Answer

Correct Answer: C
RoCE allows GPUs on different physical nodes to communicate directly with sub-microsecond latency, bypassing the host CPU entirely. Time-slicing is used for sharing a single GPU among smaller models, and the Gateway API handles inbound client HTTP traffic, not backend GPU communication.

Question 4: Investigating Access

Question: A developer is trying to query your newly published 3.3 MaaS endpoint using a curl command. They receive a 401 Unauthorized HTTP error. What is the most likely cause?

  • A. They exceeded their Tier’s token limit for the minute.

  • B. They are using their standard OpenShift ServiceAccount token instead of generating a specific MaaS API Token.

  • C. The llm-d backend pods are still pending and waiting for GPU resources.

  • D. The HTTPRoute is missing.

Click here for the Answer

Correct Answer: B
A 401 error means authentication failed at the Gateway (intercepted by Authorino). In 3.3 MaaS, standard Kubernetes tokens are not accepted; users must generate a specific MaaS token from the Gen AI Studio. Exceeding a limit results in a 429 Too Many Requests error.

Question 5: The Governance Engine

Question: Behind the scenes, which component is physically tracking the token consumption math and instructing the gateway to reject traffic when a user exceeds their Tier’s limits?

  • A. The vLLM Worker Pods

  • B. TrustyAI

  • C. Limitador (part of Kuadrant / Red Hat Connectivity Link)

  • D. The OpenShift Router

Click here for the Answer

Correct Answer: C
Limitador tracks the token consumption counters globally across the cluster. It evaluates the math against the defined Service Tiers and instructs the Envoy proxy to drop traffic (429 error) before it ever reaches the expensive vLLM pods.

Question 6: Observability

Question: When monitoring the performance of your MaaS deployment, which metric is most directly tied to the end-user’s perceived responsiveness of an LLM?

  • A. GPU Memory Utilization

  • B. Time to First Token (TTFT)

  • C. Total number of requests per second

  • D. CPU utilization of the orchestrator node

Click here for the Answer

Correct Answer: B
TTFT (Time to First Token) measures the latency from request submission until the first word is streamed back to the user. While GPU utilization and request rates are important operational metrics, TTFT dictates whether the user feels the application is "fast" or "laggy."
