Governing the AI Factory: The AI Administrator Lab (OpenShift AI 3.3)

The Platform Engineers have finished building the factory. The Gateway is open, Kueue is managing the GPUs, and Kuadrant is standing by. Now, it’s time to run the business.

In this module, we are changing personas. You are no longer writing YAML; you are the customer’s AI Administrator. Your job is to take the raw infrastructure and turn it into a governed, rate-limited "Models as a Service" product for your internal developers.

We are going to use the new Gen AI Studio dashboard to define our service tiers, deploy a model, and generate our first API token.

Step 1: Defining the Service Tiers

MaaS assigns users to service tiers based on their OpenShift group membership. Tiers are the heart of your ROI strategy—they prevent a single rogue script from exhausting the company’s GPU budget by enforcing strict token and request limits.

OpenShift AI 3.3 includes three default tiers: Free, Premium, and Enterprise. Let’s configure the Premium tier for our developers.

  1. Log in to the OpenShift AI dashboard as an administrator.

  2. In the left navigation menu, click Settings → Tiers.

  3. Click the Premium tier, then click Edit tier.

  4. Name & Level: Leave the name as Premium. The priority Level determines who wins if a user is in multiple groups. (e.g., Level 2 overrides Level 1).

  5. Groups: Select the OpenShift group that contains your developers (e.g., data-science-users or premium-api-users).

  6. Configure the Rate Limits: This is where the magic happens.

    • Select Enforce token rate limit and set it to 10,000 tokens per 1 minute.

    • Select Enforce request rate limit and set it to 100 requests per 1 minute.

  7. Click Update tier.

The Pitch: Remind your customers that Kuadrant is enforcing these limits at the network edge. If a developer exceeds 10,000 tokens, the request is dropped before it hits the GPU, saving massive amounts of compute.

Step 2: Deploying the Model (The Visual Way)

In Module 3, we deployed a model using YAML. Now, let’s see how the AI Administrator does it through the UI to publish a model to the catalog.

  1. In the left navigation menu, click Projects and select your target project (e.g., maas-production).

  2. Click the Deployments tab, then click Deploy model.

  3. Model Details: * Select Generative AI model as the Model type.

    • Specify your model location (e.g., an S3 bucket containing granite-3.0-8b-instruct). Click Next.

  4. Model Deployment:

    • Enter a unique deployment name (e.g., granite-premium).

    • Select Distributed inference with llm-d as the serving runtime.

    • Select the appropriate hardware profile (e.g., 1 NVIDIA GPU). Click Next.

  5. Advanced Settings (The MaaS Integration):

    • Select the Publish as MaaS endpoint checkbox. (This is the button that tells Kuadrant to protect the route!)

    • Under Tier access, select Specific tiers and choose Premium.

  6. Click Deploy.

Step 3: Generating Tokens and Testing the Factory

Once the model status displays a green checkmark, the factory is live. The developer can now log in, get their keys, and start building applications.

  1. In the left navigation menu, go to Gen AI Studio → AI asset endpoints → Models as a service.

  2. Locate your deployed model. You will see a "MaaS" badge next to its name. Click View in the Inference endpoint column.

  3. Click Generate API Key.

  4. Securely copy the token. (Note: This token is a JSON Web Token (JWT) tied specifically to the user and their assigned tier).

The Developer Experience

Your internal developers can now query the governed endpoint just like they would query OpenAI, but with total data sovereignty.

export MAAS_TOKEN="<your_generated_token>"
export INFERENCE_URL="https://maas.<your-cluster-domain>"

curl -k -X POST "${INFERENCE_URL}/llm/granite-premium/v1/chat/completions" \
  -H "Authorization: Bearer ${MAAS_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "granite-premium",
    "messages": [{"role": "user", "content": "Write a python script to parse JSON."}],
    "max_tokens": 500
  }'
