Phase 4a: Distributed Inference — llm-d

Value: TCO, SLOs, Sovereign AI. Distributed inference is required for MaaS.

Outcome

Associates understand why distributed inference is required for MaaS and can articulate and deploy the llm-d stack (Gateway API, Inference Scheduler, vLLM workers, request lifecycle).

Takeaway

  • Intelligent routing and pooling eliminate idle GPU waste and enable guaranteed SLOs.

  • Engineering outcome: Conceptual and hands-on grasp of llm-d so MaaS is not a black box.

Use the navigation menu to work through the llm-d module (vLLM and llm-d, Architecture, Simulator, Lab Guide, Summary).

Checkpoint

  • llm-d stack deployed or Arcade lab completed; request flow (Gateway → Scheduler → vLLM) understood.
