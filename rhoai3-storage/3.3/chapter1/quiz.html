Knowledge Check: Verify Your Architecture Skills

Did you master the factory floor? Let’s find out.

Question 1: The Pipeline Strategy

Scenario: You are architecting a new AI Pipeline. The tasks need to pass a massive 50GB intermediate dataset between each other. What is the most efficient storage strategy in OpenShift AI 3.3 to handle this data transfer?

  • A. Upload the dataset to an external S3 bucket at the end of each task and download it at the start of the next.

  • B. Use an Ephemeral Pipeline Workspace backed by a Persistent Volume Claim (PVC).

  • C. Store the data inside the internal MySQL database.

  • D. Package the intermediate data into an OCI container image.

Answer: B. OpenShift AI 3.3 introduces Pipeline Workspaces. This allows pipeline components to exchange large intermediate data directly via local cluster disk, improving performance and reducing object storage costs.

Question 2: The Security

Scenario: You need to rotate the AWS Access Keys used by the "Fraud Detection" team. How do you update the credentials without breaking the data scientist’s code?

  • A. Email the new keys to the team and ask them to update their Jupyter notebooks.

  • B. Update the hardcoded values in the deployment.yaml file.

  • C. Update the Kubernetes Secret associated with the Data Connection.

  • D. Delete the workbench and recreate it.

Answer: C. Data Connections abstract credentials into Kubernetes Secrets. Updating the Secret automatically propagates the new values to the environment variables.

Question 3: The Architecture (GitOps)

Scenario: Your organization requires that all production AI Pipelines be managed via GitOps (e.g., using Argo CD). How must you configure your pipeline server to support this?

  • A. Set the pipeline server to store definitions in the internal MariaDB database.

  • B. Export the pipelines manually as ZIP files and upload them to Git.

  • C. Configure the pipeline server to use Kubernetes API storage (spec.apiServer.pipelineStore: kubernetes).

  • D. Pipelines cannot be managed by GitOps tools.

Answer: C. By using the Kubernetes API storage mode, pipelines and pipeline versions are stored as standard Custom Resources (Pipeline and PipelineVersion) in the cluster, making them natively compatible with GitOps tools.

Question 4: The Operational Hazard

Scenario: You are deploying a 70GB Foundation Model using the oci:// protocol (Modelcar). The network is healthy, but the deployment repeatedly fails and marks the pod as "Failed" after exactly 10 minutes. What is the most likely cause?

  • A. The CRI-O node cache is completely full.

  • B. The OCI registry requires a different connection type.

  • C. The deployment exceeded the default Knative progress-deadline timeout.

  • D. The Persistent Volume Claim (PVC) is set to ReadOnlyMany.

Answer: C. Pulling massive multi-gigabyte container images can easily exceed the default 10-minute timeout configured in underlying Knative Serving objects. You must explicitly annotate the InferenceService to increase this threshold.
