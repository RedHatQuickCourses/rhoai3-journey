The Architecture of Trust

To industrialize AI, you must move from "monitoring" to "intercepting."

In a traditional MLOps pipeline, monitoring happens after the fact. You analyze logs at the end of the week to see if your model drifted.

In the Industrial AI Factory, this is too late. If a GenAI model leaks PII or outputs hate speech, the damage is instantaneous. Therefore, the architecture of Red Hat OpenShift AI (RHOAI) 3.x shifts from passive observation to active interception.

This module deconstructs the technical stack you will be building. It explains how TrustyAI sits between your users and your models to enforce the "Zero Trust" paradigm.

The Core Architecture: The "Man-in-the-Middle"

TrustyAI is not a single binary; it is a set of distinct services that integrate with the KServe inference stack. Understanding these components is critical for the "Systems Architect" persona.

1. The Guardrails Orchestrator (The "Shield")

This is the active defense layer. It functions as a proxy that intercepts every inference request.

  • Role: It pauses the request, sends the prompt to a series of "Detectors" (e.g., Regex, Toxicity Model, PII Scanner), and waits for a verdict.

  • The Decision:

    • Pass: The request is forwarded to the LLM.

    • Block: The request is rejected immediately. The LLM never sees it, saving GPU compute costs.

  • Deployment: Runs as a sidecar or gateway service within the Service Mesh, ensuring mTLS encryption.

2. The Model Serving Runtime (The "Engine")

This is where the actual intelligence lives (e.g., Granite-8B, Llama-3).

  • Role: Pure inference. In this architecture, the model is "blind" to the outside world; it only accepts traffic from the Guardrails Orchestrator.

  • Technology: We use vLLM on GPU nodes to handle the heavy lifting.

3. The TrustyAI Service (The "Auditor")

This is the passive observation layer. It asynchronously consumes payload data to calculate long-term metrics.

  • Role: It computes "drift" (how much data has changed over time) and "fairness" (bias detection).

  • Storage: It writes these metrics to Prometheus for visualization in Grafana or the RHOAI dashboard.
