Module 1.2: Fast-Track Environment Setup

To guard a model, you first need a model.

TrustyAI does not run in a vacuum. It requires a live AI workload to monitor. In a real-world scenario, you might have an existing KServe or vLLM deployment. In this lab, we will build a dedicated "Sandbox" environment from scratch in less than 5 minutes.

What this setup does

We will use two automated scripts to:

  1. Create a namespace called trustyai-lab.

  2. Deploy MinIO (S3-compatible storage) to act as our "Model Vault."

  3. Configure the RHOAI Data Connection secrets automatically.

  4. Run a Kubernetes Job to download Granite 3.2 2B Instruct from Hugging Face and upload it to your private MinIO vault.

Step 1: Infrastructure Setup (setup.sh)

This script handles the "plumbing." It creates the project and deploys the MinIO storage server.

Create the setup script
cat <<EOF > setup.sh
#!/bin/bash
set -e

# CONFIGURATION
NAMESPACE="trustyai-lab"
MINIO_ACCESS_KEY="minio"
MINIO_SECRET_KEY="minio123"

echo "🚀 Starting TrustyAI Lab Infrastructure..."

# 1. Create Namespace
if ! oc get project "\$NAMESPACE" > /dev/null 2>&1; then
    echo "➤ Creating namespace \$NAMESPACE..."
    oc new-project "\$NAMESPACE"
else
    echo "✔ Namespace \$NAMESPACE exists."
fi

# 2. Deploy MinIO (The Vault)
echo "➤ Deploying MinIO..."
# Note: In a real lab, ensure these YAMLs exist.
# For this fast-track, we assume standard MinIO deployment files are present
# or we use a simple helm/oc apply approach.
oc apply -f deploy/infrastructure/minio/ -n "\$NAMESPACE" || echo "⚠️  Make sure MinIO YAMLs are in 'deploy/infrastructure/minio/'"

# 3. Create Data Connection Secrets (Required by KServe)
echo "➤ Wiring Data Connections..."
oc create secret generic storage-config \\
    --from-literal=AWS_ACCESS_KEY_ID="\$MINIO_ACCESS_KEY" \\
    --from-literal=AWS_SECRET_ACCESS_KEY="\$MINIO_SECRET_KEY" \\
    --from-literal=AWS_S3_ENDPOINT="http://minio-service.\$NAMESPACE.svc.cluster.local:9000" \\
    --from-literal=AWS_DEFAULT_REGION="us-east-1" \\
    --from-literal=AWS_S3_BUCKET="models" \\
    -n "\$NAMESPACE" \\
    --dry-run=client -o yaml | oc apply -f -

# Label for RHOAI Dashboard Visibility
oc label secret storage-config "opendatahub.io/dashboard=true" "opendatahub.io/managed=true" -n "\$NAMESPACE" --overwrite

# Create 'models' secret for the ingestion job
oc create secret generic models \\
    --from-literal=AWS_ACCESS_KEY_ID="\$MINIO_ACCESS_KEY" \\
    --from-literal=AWS_SECRET_ACCESS_KEY="\$MINIO_SECRET_KEY" \\
    --from-literal=AWS_S3_ENDPOINT="http://minio-service.\$NAMESPACE.svc.cluster.local:9000" \\
    --from-literal=AWS_DEFAULT_REGION="us-east-1" \\
    --from-literal=AWS_S3_BUCKET="models" \\
    -n "\$NAMESPACE" \\
    --dry-run=client -o yaml | oc apply -f -

echo "✅ Infrastructure Ready in namespace: \$NAMESPACE"
EOF
Run the setup
chmod +x setup.sh
./setup.sh

Step 2: Model Ingestion (fast-track.sh)

Now that MinIO is running, we need to populate it. We will use the Granite 3.2 2B Instruct model because it is lightweight, fast to download, and powerful enough to demonstrate complex reasoning and toxicity risks.

Create the ingestion script
cat <<EOF > fast-track.sh
#!/bin/bash
set -e

# CONFIGURATION
NAMESPACE="trustyai-lab"
MODEL_ID="ibm-granite/granite-3.2-2b-instruct"
S3_BUCKET="models"
S3_FOLDER="granite-3.2-2b"
SERVICE_ACCOUNT="model-loader-sa"

echo "🚀 Starting Model Ingestion for TrustyAI..."
echo "🎯 Target: \$MODEL_ID -> s3://\$S3_BUCKET/\$S3_FOLDER"

# 1. Create Service Account for the Job
oc create sa \$SERVICE_ACCOUNT -n "\$NAMESPACE" --dry-run=client -o yaml | oc apply -f -
oc adm policy add-scc-to-user anyuid -z \$SERVICE_ACCOUNT -n "\$NAMESPACE" > /dev/null 2>&1

# 2. Define Python Ingestion Logic
cat <<PY > /tmp/ingest_script.py
import os
import boto3
from huggingface_hub import snapshot_download
from botocore.client import Config

MODEL_ID = "${MODEL_ID}"
BUCKET = "${S3_BUCKET}"
FOLDER = "${S3_FOLDER}"

def main():
    print(f"Downloading {MODEL_ID}...")
    # Download specific extensions to keep image size small
    local_dir = snapshot_download(repo_id=MODEL_ID, allow_patterns=["*.json", "*.safetensors", "*.model", "tokenizer*"])

    print(f"Uploading to MinIO ({BUCKET}/{FOLDER})...")
    s3 = boto3.client('s3',
                      endpoint_url=os.getenv("AWS_S3_ENDPOINT"),
                      aws_access_key_id=os.getenv("AWS_ACCESS_KEY_ID"),
                      aws_secret_access_key=os.getenv("AWS_SECRET_ACCESS_KEY"),
                      config=Config(signature_version='s3v4'))

    try: s3.create_bucket(Bucket=BUCKET)
    except: pass

    for root, _, files in os.walk(local_dir):
        for file in files:
            local_path = os.path.join(root, file)
            rel_path = os.path.relpath(local_path, local_dir)
            s3_key = os.path.join(FOLDER, rel_path)
            print(f" -> Uploading {s3_key}")
            s3.upload_file(local_path, BUCKET, s3_key)

if __name__ == "__main__":
    main()
PY

# 3. Create ConfigMap with the script
oc create configmap ingestion-code --from-file=/tmp/ingest_script.py -n "\$NAMESPACE" --dry-run=client -o yaml | oc apply -f -

# 4. Submit Kubernetes Job
oc delete job model-loader -n "\$NAMESPACE" --ignore-not-found
cat <<YAML | oc apply -f -
apiVersion: batch/v1
kind: Job
metadata:
  name: model-loader
  namespace: \$NAMESPACE
spec:
  backoffLimit: 1
  template:
    spec:
      serviceAccountName: \$SERVICE_ACCOUNT
      containers:
      - name: loader
        image: registry.access.redhat.com/ubi9/python-311:latest
        command: ["/bin/bash", "-c"]
        args:
          - |
            pip install boto3 huggingface-hub --quiet --no-cache-dir && \
            python /scripts/ingest_script.py
        env:
        - name: AWS_ACCESS_KEY_ID
          valueFrom: { secretKeyRef: { name: models, key: AWS_ACCESS_KEY_ID } }
        - name: AWS_SECRET_ACCESS_KEY
          valueFrom: { secretKeyRef: { name: models, key: AWS_SECRET_ACCESS_KEY } }
        - name: AWS_S3_ENDPOINT
          valueFrom: { secretKeyRef: { name: models, key: AWS_S3_ENDPOINT } }
        volumeMounts:
        - name: code
          mountPath: /scripts
      restartPolicy: Never
      volumes:
      - name: code
        configMap:
          name: ingestion-code
YAML

echo "⏳ Ingestion Job Submitted. Run 'oc logs job/model-loader -n \$NAMESPACE -f' to monitor."
EOF
Run the ingestion
chmod +x fast-track.sh
./fast-track.sh

Step 3: Verify Success

Wait for the job to complete (approx. 2-5 minutes).

oc wait --for=condition=complete job/model-loader -n trustyai-lab --timeout=300s

You now have: 1. Project: trustyai-lab 2. Storage: MinIO with ibm-granite/granite-3.2-2b-instruct in the models bucket. 3. Connection: A storage-config secret ready for KServe.

Environment is ready. Proceed to Module 1.1: Deploying the Shield to install TrustyAI.
