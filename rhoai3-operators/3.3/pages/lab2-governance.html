Lab 2 — Configuring Hardware Profiles & Accelerators

Your GPUs are online, but your data scientists cannot see them. It is time to connect the factory floor to the storefront.

In Lab 0, we deployed the Node Feature Discovery (NFD) and NVIDIA GPU operators. In Lab 1, we activated the core AI factory subsystems. At this point, your OpenShift cluster is fully aware of its hardware acceleration capabilities. However, your data scientists are not Kubernetes engineers—they should not be writing nodeSelectors or tolerations manually to schedule a model training job.

We bridge this gap using Hardware Profiles. A Hardware Profile acts as a product catalog entry, translating complex Kubernetes scheduling constraints into a simple, user-friendly dropdown menu in the OpenShift AI Dashboard.

Technology Preview Notice

The hardware profiles feature is currently available in Red Hat OpenShift AI 3.3 as a Technology Preview feature. Technology Preview features provide early access to upcoming product features , but are not recommended for production environments requiring strict Red Hat SLAs.

1. The Strategy: Hiding the Infrastructure

When a data scientist requests a GPU for a workbench or a model deployment, the underlying pod must contain specific resources.limits and tolerations to ensure it lands on a node with the physical hardware.

If you force users to manually inject: nvidia.com/gpu: 1 and tolerations: nvidia.com/gpu:NoSchedule …​you will spend your entire week debugging pending pods due to typographical errors.

By defining a HardwareProfile Custom Resource (CR) as the platform engineer, you encode these rules once.

2. Verify Accelerator Detection

Before we create the profile, let’s mathematically prove the Operator layer did its job. We need to ensure the cluster actually recognizes the allocatable GPU capacity.

In your Web Terminal, check a specific GPU-enabled worker node:

# Expected output when the accelerator is detected correctly
oc describe node <your-gpu-node-name> | grep -A 5 "Allocatable:"

You must see the accelerator listed under the allocatable resources, for example: nvidia.com/gpu: 1.

If this reads 0 or is missing, your NVIDIA GPU Operator ClusterPolicy from Lab 0 has not finished reconciling, or NFD failed to label the node. Do not proceed until you have allocatable GPU capacity.

3. Applying the Hardware Profile Blueprint

We will use our declarative, GitOps-friendly approach to create a standard NVIDIA GPU profile.

Copy and paste the following into your authenticated Web Terminal. This manifest creates a profile that automatically injects the exact identifiers and tolerations required by the NVIDIA GPU Operator into any workload that selects it:

cat <<EOF | oc apply -f -
apiVersion: dashboard.opendatahub.io/v1alpha1
kind: HardwareProfile
metadata:
  name: nvidia-gpu-standard
  namespace: redhat-ods-applications
spec:
  displayName: "Standard NVIDIA GPU"
  description: "Allocates 1 NVIDIA GPU for compute-intensive workloads and model serving."
  enabled: true
  identifiers:
    - identifier: nvidia.com/gpu
      displayName: NVIDIA GPU
      minCount: 1
      maxCount: 4
      defaultCount: 1
  tolerations:
    - effect: NoSchedule
      key: nvidia.com/gpu
      operator: Exists
EOF

4. Verification & Validation

Let’s verify that the catalog item is now available for your tenants.

Step A: Check the Custom Resource

Confirm the cluster accepted the profile:

oc get hardwareprofiles -n redhat-ods-applications

Expect to see nvidia-gpu-standard listed.

Step B: Verify the User Experience

  1. Open the OpenShift AI Dashboard in your browser.

  2. Navigate to Settings → Hardware profiles to view the active catalog.

  3. To test the execution path, navigate to Projects → Create workbench].

  4. Scroll down to the Deployment size section.

  5. You will now see a Hardware profile list. Open it and confirm that Standard NVIDIA GPU is available for selection.

By selecting this profile, the user’s workbench pod will automatically request the nvidia.com/gpu limits and tolerate the GPU node taints. The factory floor is now officially connected to the storefront.
