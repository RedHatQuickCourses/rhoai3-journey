Quiz: Model Selection Criteria

Estimated reading time: 10 minutes.

Objective

Test understanding of model selection criteria through a knowledge check covering context length limitations, tool calling considerations, and choosing the right model type for specific workloads.

Knowledge Check

Answer the following questions to test your understanding of model selection:

  1. Which model architecture typically provides higher throughput per GPU?

    1. Dense models (all parameters active)

    2. Sparse models (Mixture of Experts)

    3. Both provide identical throughput

    4. It depends only on batch size

  2. What is the primary advantage of reasoning models?

    1. Faster inference speed

    2. Lower cost per query

    3. Better performance on complex tasks requiring multi-step thinking

    4. Smaller memory footprint

  3. A customer needs a chatbot for simple product recommendations with <5 second response time. Which model type is most appropriate?

    1. Reasoning model (for accuracy)

    2. Non-reasoning model (for speed and cost)

    3. Sparse Mixture of Experts

    4. The largest available model

  4. What is a key consideration when deploying Mixture of Experts models?

    1. They require less total GPU memory than dense models

    2. They need complex expert sharding across GPUs

    3. They always provide lower latency

    4. They cannot be quantized

  5. Context length primarily affects:

    1. Model accuracy only

    2. GPU memory requirements for KV cache

    3. Model training time

    4. Number of parameters in the model

  6. When would you choose a dense model over a sparse model?

    1. When you need maximum throughput

    2. When you want predictable latency and simpler deployment

    3. When serving millions of requests per day

    4. When you have unlimited GPU resources

  7. Tool calling capability in models is most important for:

    1. Simple text generation tasks

    2. Agentic workflows that need to interact with external APIs and databases

    3. Reducing inference costs

    4. Improving model accuracy

  8. If a reasoning model generates 800 tokens per query and a non-reasoning model generates 150 tokens, the reasoning model will:

    1. Have the same total latency

    2. Occupy the GPU ~5x longer per request

    3. Be faster due to better optimization

    4. Use less memory

Answer Key

  1. b - Sparse (MoE) models provide higher throughput by activating fewer parameters per token

  2. c - Reasoning models excel at complex, multi-step tasks despite higher cost

  3. b - Non-reasoning model provides fast, cost-effective responses for simple tasks

  4. b - MoE models require careful expert sharding across GPUs for optimal performance

  5. b - Longer context requires more GPU memory for the KV cache

  6. b - Dense models offer predictable latency and are easier to deploy

  7. b - Tool calling enables models to act as agents interacting with external systems

  8. b - More tokens means longer generation time and GPU occupation
