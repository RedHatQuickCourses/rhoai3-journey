Lab: Consuming the OpenAI-Compatible API

Estimated reading time: 25 minutes.

Objective

Use curl and Python to interact with the /v1/chat/completions endpoint using Bearer token authentication. Read response headers to identify remaining tokens and experiment with setting max tokens. Test API key authentication and rate limiting behavior.

Not started yet.

Before you begin

Ensure you have:

  • An API key generated from the MaaS dashboard

  • The MaaS base URL for your cluster

  • curl and Python 3.8+ installed

  • Access to a model published as a MaaS endpoint

Instructions

Step 4: Python Client with Error Handling

Create a Python script with proper error handling:

import os
import time
from openai import OpenAI, RateLimitError, AuthenticationError

# Initialize client
client = OpenAI(
    api_key=os.environ.get("MAAS_API_KEY"),
    base_url=os.environ.get("MAAS_BASE_URL")
)

def chat_with_retry(prompt, max_retries=3):
    """Make a chat completion request with retry logic."""
    for attempt in range(max_retries):
        try:
            response = client.chat.completions.create(
                model="nemotron-3-nano",
                messages=[
                    {"role": "user", "content": prompt}
                ],
                max_tokens=200,
                temperature=0.7
            )
            return response

        except RateLimitError as e:
            if attempt < max_retries - 1:
                wait_time = 2 ** attempt  # Exponential backoff
                print(f"Rate limit hit. Waiting {wait_time}s before retry...")
                time.sleep(wait_time)
            else:
                print("Max retries reached. Rate limit still in effect.")
                raise

        except AuthenticationError:
            print("Authentication failed. Check your API key.")
            raise

        except Exception as e:
            print(f"Unexpected error: {e}")
            raise

# Example usage
if __name__ == "__main__":
    response = chat_with_retry("Explain Kubernetes Operators briefly.")

    print("Response:", response.choices[0].message.content)
    print(f"\nTokens used: {response.usage.total_tokens}")
    print(f"  - Prompt: {response.usage.prompt_tokens}")
    print(f"  - Completion: {response.usage.completion_tokens}")

Run the script:

Step 6: Monitor Response Headers with Python

Access rate limit headers programmatically:

import httpx

def get_rate_limit_info():
    """Fetch and display rate limit information."""
    headers = {
        "Authorization": f"Bearer {os.environ['MAAS_API_KEY']}",
        "Content-Type": "application/json"
    }

    payload = {
        "model": "nemotron-3-nano",
        "messages": [{"role": "user", "content": "test"}],
        "max_tokens": 10
    }

    response = httpx.post(
        f"{os.environ['MAAS_BASE_URL']}/chat/completions",
        json=payload,
        headers=headers
    )

    print("Rate Limit Info:")
    print(f"  Limit: {response.headers.get('X-RateLimit-Limit')}/min")
    print(f"  Remaining: {response.headers.get('X-RateLimit-Remaining')}")
    print(f"  Reset at: {response.headers.get('X-RateLimit-Reset')}")
    print(f"  Tier: {response.headers.get('X-RateLimit-Tier')}")

    return response.json()

result = get_rate_limit_info()

Step 7: Test Rate Limiting Behavior

Intentionally trigger rate limiting to see error handling:

def test_rate_limiting():
    """Test rate limit by making rapid requests."""
    print("Testing rate limiting (making rapid requests)...")

    for i in range(150):  # Exceed premium tier limit of 100/min
        try:
            response = client.chat.completions.create(
                model="nemotron-3-nano",
                messages=[{"role": "user", "content": "test"}],
                max_tokens=5
            )
            print(f"Request {i+1}: Success")

        except RateLimitError as e:
            print(f"Request {i+1}: RATE LIMITED")
            print(f"Error details: {e}")

            # Parse reset time from error
            if hasattr(e, 'response') and e.response:
                reset_time = e.response.headers.get('X-RateLimit-Reset')
                print(f"Rate limit resets at: {reset_time}")

            break

test_rate_limiting()

Step 8: Streaming Responses

Test streaming for real-time token generation:

def stream_chat(prompt):
    """Stream chat completion tokens as they're generated."""
    response = client.chat.completions.create(
        model="nemotron-3-nano",
        messages=[{"role": "user", "content": prompt}],
        max_tokens=150,
        stream=True  # Enable streaming
    )

    print("Streaming response:")
    for chunk in response:
        if chunk.choices[0].delta.content:
            print(chunk.choices[0].delta.content, end='', flush=True)
    print()  # Newline after streaming completes

stream_chat("Explain the benefits of MaaS in three bullet points.")

You have successfully consumed MaaS APIs using curl and Python!

Best Practices Summary

  • Always handle rate limits with exponential backoff

  • Monitor token usage to optimize costs

  • Use environment variables for API keys

  • Implement error handling for 401, 403, 429, 500, 503

  • Check response headers for rate limit information

  • Set appropriate max_tokens to control costs and latency

  • Use streaming for better user experience in interactive apps
