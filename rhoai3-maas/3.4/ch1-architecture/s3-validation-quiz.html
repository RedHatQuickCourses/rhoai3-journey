Quiz: Architecture Validation

Estimated reading time: 10 minutes.

Objective

Test recall of architectural layers and components through a knowledge check quiz covering the GPU Silo problem, MaaS governance benefits, and the foundational GPU operator stack.

Knowledge Check

Answer the following questions to test your understanding of MaaS architecture:

  1. What is the primary problem that MaaS solves?

    1. Slow model inference speeds

    2. GPU hardware trapped in silos leading to idle capacity and bottlenecks

    3. Lack of AI models in the organization

    4. Difficulty deploying containers

  2. Which component is responsible for rate limiting in the MaaS architecture?

    1. KServe

    2. Gateway API

    3. Limitador (part of Kuadrant)

    4. NVIDIA GPU Operator

  3. What is the role of the NVIDIA GPU Operator in MaaS?

    1. Provides model serving capabilities

    2. Manages API authentication

    3. Provides foundational GPU drivers, DCGM, container toolkit, and MIG management

    4. Handles rate limiting

  4. Which component provides the model serving framework?

    1. Kuadrant

    2. KServe

    3. Authorino

    4. Gateway API

  5. What does Authorino (part of Kuadrant) provide?

    1. GPU driver management

    2. Model versioning

    3. JWT validation and authentication

    4. Container orchestration

  6. Which layer sits at the foundation of the MaaS architecture?

    1. MaaS Governance Layer

    2. Foundational Accelerator Enablement (GPU Operator)

    3. Application Layer

    4. Network Layer

Answer Key

  1. b - MaaS solves the GPU Silo problem where expensive hardware is underutilized

  2. c - Limitador enforces rate limits based on service tiers

  3. c - GPU Operator provides the foundational GPU infrastructure

  4. b - KServe is the Kubernetes-native model serving framework

  5. c - Authorino handles JWT validation and authentication

  6. b - The GPU Operator layer forms the foundation
