Course Overview

The Problem We're Solving

The GPU Crisis

Organizations today face a critical challenge: expensive, limited GPU hardware trapped within individual teams, leading to simultaneous idle capacity and project bottlenecks.

❌ Without Centralized Approach

  • Teams request dedicated GPUs
  • Hardware sits under-utilized
  • High-priority projects stall
  • No usage quotas or governance
  • Lack of multi-tenant security
  • No standardized model library

✅ With Models as a Service (MaaS)

  • Centralized governance layer
  • Efficient GPU resource sharing
  • Enforced usage quotas
  • Multi-tenant security
  • Standardized model catalog
  • Orchestration and SLOs

The Traditional Timeline Problem

The Question: How fast can you move from "we want AI" to "AI delivers business value"?

Industry Reality: Most organizations struggle with 6–18 months of infrastructure setup before development begins.

  • GPU costs are high
  • Governance is complex
  • Knowledge is distributed
  • Integration is challenging

Our Solution

Progressive, Opinionated Architecture

This course provides a starting point—not the final destination. We show you:

  • The business decision behind each component
  • The technical implementation with working code
  • No fluff—result-focused instruction
  • Extensible foundation for your production needs

Using the Course Offline

Why Offline Access?

  • Air-gapped environments: Many enterprise OpenShift clusters are isolated from the internet
  • Unreliable connectivity: Work without depending on network availability
  • Custom modifications: Add company-specific content and branding
  • Internal distribution: Host on internal servers or share via USB drives

Quick Start: Offline Setup

Simple Option: Download the pre-built site from GitHub Pages

  1. Visit the published site:
    https://redhatquickcourses.github.io/rhoai3-journey/
  2. Use browser "Save Page As" → "Webpage, Complete"
  3. Or use wget to mirror the site:
    wget --mirror --convert-links --adjust-extension --page-requisites --no-parent https://redhatquickcourses.github.io/rhoai3-journey/
  4. Open the local index.html in any browser

Advanced: Build Your Own Offline Copy

For customization or the latest content:

Prerequisites

  • Git (to clone repositories)
  • Node.js 16+ and npm (for local builds)
  • OR Docker (for containerized builds)

Method 1: Clone All Repositories

# Create a parent directory
mkdir rhoai3-offline
cd rhoai3-offline

# Clone the journey repository
git clone https://github.com/RedHatQuickCourses/rhoai3-journey.git

# Clone all 10 content repositories
git clone https://github.com/RedHatQuickCourses/rhoai3-operators.git
git clone https://github.com/RedHatQuickCourses/rhoai3-storage.git
git clone https://github.com/RedHatQuickCourses/rhoai3-registry.git
git clone https://github.com/RedHatQuickCourses/rhoai3-catalog.git
git clone https://github.com/RedHatQuickCourses/rhoai3-hwprofiles.git
git clone https://github.com/RedHatQuickCourses/rhoai3-gpu-aas.git
git clone https://github.com/RedHatQuickCourses/rhoai3-deploy.git
git clone https://github.com/RedHatQuickCourses/rhoai3-llmd.git
git clone https://github.com/RedHatQuickCourses/rhoai3-maas.git
git clone https://github.com/RedHatQuickCourses/rhoai3-validate.git

# Build the site
cd rhoai3-journey
npm install
npx antora antora-playbook.yml

# Site is now in: build/site/
# Open build/site/index.html in a browser

Method 2: Let Antora Fetch Repositories (Internet Required)

# Clone only the journey repository
git clone https://github.com/RedHatQuickCourses/rhoai3-journey.git
cd rhoai3-journey

# Build with --fetch (Antora will clone other repos to cache)
npm install
npx antora --fetch antora-playbook.yml

# Site is in: build/site/

Method 3: Docker Build (No Node.js Required)

# Clone repositories as in Method 1, then:
cd rhoai3-offline

docker run -u $(id -u) -v $PWD:/workspace:Z --rm -t \
  -w /workspace/rhoai3-journey \
  antora/antora antora-playbook.yml

# Site is in: rhoai3-journey/build/site/

Distributing the Offline Site

USB Drive

Copy the build/site/ directory to a USB drive. Recipients open index.html in any browser.

Internal Web Server

Host on Apache, Nginx, or any web server. Point to build/site/ as the document root.

Compressed Archive

Create a zip file: zip -r rhoai3-journey.zip build/site/ and share via email or file share.

Local Python Server

For quick testing: cd build/site && python3 -m http.server 8080

Customizing the Course

Common Customization Scenarios

1. Add Company Branding

Modify the UI bundle to include your company logo and colors:

  • Edit supplemental-ui/partials/header-content.hbs for header logo
  • Edit supplemental-ui/css/custom.css for custom styles
  • Replace supplemental-ui/img/logo.png with your logo

2. Add Internal Notes or Warnings

Insert company-specific content in phase introduction pages:

// Edit: modules/ROOT/pages/phase1.adoc

[WARNING]
====
Internal Note: For <YourCompany> deployments,
ensure you use the approved storage class:
`company-approved-storage-class`
====

3. Remove or Reorder Phases

Edit modules/ROOT/nav2.adoc to change navigation order or hide sections:

// Comment out sections you don't want:
// * 4: Quality and SLOs (validate)
// ** xref:phase4.adoc[Phase overview]

4. Add Custom Lab Environment Details

Create a custom intro page with your cluster details:

// Create: modules/ROOT/pages/company-intro.adoc

= Lab Environment for <YourCompany>

== Cluster Access
* Console: https://console.yourcluster.com
* API: https://api.yourcluster.com:6443

== Credentials
Contact IT help desk for credentials.

== Support
Internal support: #ai-platform-support

Then add to navigation:

// In nav2.adoc
* xref:company-intro.adoc[Company Lab Environment]

5. Bundle with Custom Tools or Scripts

Add company-specific automation scripts:

  • Place scripts in supplemental-ui/downloads/
  • Reference in pages: link:{attachmentsdir}/setup-company-env.sh[Download Setup Script]

Rebuilding After Customization

npm run build

Test locally:

npm run serve

Then open http://localhost:8080 to verify changes.

Multi-Repository Architecture

Understanding the Structure: This is NOT a single course—it's a meta-course that aggregates content from 10 separate repositories.

How It Works

1. Journey Repository

Contains only navigation structure and phase overview pages

2. Content Repositories

10 separate repos with actual course content (presentations, labs, quizzes)

3. Antora Build System

Aggregates all repos into a single unified website

4. Static Site Output

Self-contained HTML site that works offline

Directory Structure

rhoai3-offline/ # Parent directory
├── rhoai3-journey/ # Journey map
│ ├── antora-playbook.yml # Lists all 10 content repos
│ ├── antora.yml # Journey component config
│ ├── modules/ROOT/ # Journey structure
│ │ ├── nav2.adoc # Master navigation
│ │ └── pages/ # Phase intro pages
│ └── build/site/ # Built site (after build)
│
├── rhoai3-operators/ # Phase 0 content
├── rhoai3-storage/ # Phase 1 content
├── rhoai3-registry/ # Phase 1 content
├── rhoai3-catalog/ # Phase 1 content
├── rhoai3-hwprofiles/ # Phase 1 content
├── rhoai3-gpu-aas/ # Phase 1 content
├── rhoai3-deploy/ # Phase 2 content
├── rhoai3-llmd/ # Phase 3 content
├── rhoai3-maas/ # Phase 3 content
└── rhoai3-validate/ # Phase 4 content

Content Aggregation

The journey repository uses Antora's include:: directive to pull content:

// File: rhoai3-journey/modules/ROOT/pages/p1-storage.adoc
// Pulls content from rhoai3-storage repository

include::rhoai3-storage:chapter1:index.adoc[]

Build & Deploy Options

Development Build (with live reload)

For active content development:

# Terminal 1 - Auto-rebuild on changes
npm run watch:adoc

# Terminal 2 - Serve locally
npm run serve

# Open: http://localhost:8080

Production Build

For final offline distribution:

npm run build

Output: build/site/ directory with complete static site

GitHub Pages (Automatic)

When you push to main branch, GitHub Actions automatically:

  1. Clones all 10 content repositories
  2. Builds the unified site with Antora
  3. Publishes to GitHub Pages

Live Site: https://<org>.github.io/rhoai3-journey/

PDF Generation

Generate a PDF version for offline reading:

npm run build:pdf

Output: build/assembler-pdf/*.pdf

Troubleshooting

Common Issues

  • Build fails: Ensure all 10 sibling repos are cloned (for local builds without --fetch)
  • Missing content: Check that antora-playbook.yml references all content sources
  • Navigation broken: Verify xref paths in nav2.adoc match actual page files
  • Images not loading: Check image paths and ensure they're in module images/ directories
